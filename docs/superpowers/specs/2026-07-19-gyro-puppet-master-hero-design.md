# Gyro Puppet Master Hero

## Goal

Replace the current right-weighted realistic hero with a centered, manga-influenced Gyro Zeppeli who appears to control the AI systems presented below the banner.

## Composition

- Keep the existing wide `1600 x 533` README hero format.
- Place Gyro on the exact horizontal center, framed from the waist up.
- Give him a calm, dominant pose with both arms extended diagonally downward.
- Show a single Steel Ball suspended between or just below his hands.
- Arrange restrained terminal panes, agent nodes, neural diagrams, and code streams beneath his hands so they read as systems he is controlling.
- Preserve enough dark negative space around the silhouette for the banner to remain readable at mobile width.

## Character Direction

- Use a youthful, elegant manga interpretation rather than a muscular photorealistic man.
- Preserve Gyro's recognizable hat and goggles, long blond hair, sharp eyes, facial markings, and confident expression.
- Use clean ink contours, selective cross-hatching, and colored manga rendering.
- Avoid heavy makeup, bodybuilder proportions, leathery realism, exaggerated grit, or generic cyberpunk styling.

## Visual Language

- Retain the blacksite and old-terminal atmosphere of the current profile.
- Use near-black, muted teal, CRT green, and small cyan highlights.
- Keep glow localized to the Steel Ball and controlled systems.
- No text, logos, watermark, excessive neon, religious symbols, or visual clutter.

## Scope

- Replace `assets/gyro-blacksite.webp` with the approved generated asset.
- Update the hero alt text in `README.md` only if the new composition makes the existing wording inaccurate.
- Do not change any other README copy, badges, sections, links, or repository files.

## Acceptance Checks

- Gyro is visually centered and remains legible in GitHub's desktop and mobile README layouts.
- The pose clearly communicates control over the systems below him.
- The face and body read as manga Gyro rather than a photorealistic action hero.
- The final asset is WebP, `1600 x 533`, reasonably compressed, and contains no text or generation artifacts.
- GitHub Markdown renders successfully with no horizontal overflow at `390px`.
